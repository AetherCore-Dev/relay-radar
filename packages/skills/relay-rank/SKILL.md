# /relay-rank — 中转站综合排名

## 评分体系

五个维度，加权计算总分（满分100）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 🏎️ 延迟 | 20% | TTFT、P50/P95延迟、吞吐量 |
| 🛡️ 稳定性 | 20% | 错误率、流式支持、连续可用性 |
| 🔬 真实性 | 25% | 模型验真得分（最高权重） |
| 💰 计费 | 20% | 价格合理性、Token计数准确性 |
| 🔍 透明度 | 15% | 价格公示、状态页、退款政策 |

## 评级标准

| 总分 | 评级 | 说明 |
|------|------|------|
| 80+ | 🌟 推荐 | 各方面表现优秀 |
| 60-79 | 👍 可用 | 基本可靠，有小问题 |
| 40-59 | 🟡 一般 | 有明显短板 |
| <40 | 👎 不推荐 | 问题严重 |

## 特殊标记

- 🚫 **模型造假** — 验证发现模型与声称不符
- ⚠️ **计费欺诈** — 检测到Token虚增或隐藏收费
- 💉 **Prompt注入** — 检测到隐藏System Prompt

## 使用方法

### 快速排名
```bash
npx relay-radar rank
```

### 完整工作流

1. 配置中转站列表 (`relay-radar.json`)
2. 运行 `relay-radar rank`
3. 查看排名表格和详细报告
4. 根据报告选择最适合的中转站

## 报告输出

生成两种格式：
- **终端表格** — 直观的排名展示
- **JSON报告** — 可用于网站展示或持续监控

## 持续监控

建议每周运行一次排名，追踪变化趋势：
```bash
# 使用cron定期运行
0 9 * * 1 cd ~/my-project && npx relay-radar rank
```
